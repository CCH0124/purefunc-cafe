# WS
* 全雙工
* 支援 binary
* 資源足夠的前提，效能較好
* 支援度 (Client, Server, Infra)
* 自行實作心跳跟斷線重連
* auth 支援度
* 傳輸量

# SSE
* 單向
* 不支援 binary
* 資源足夠的前提，效能有限制
* Base on HTTP
* 內建心跳和斷線重連
* auth 支援度
* 傳輸量
